## 使用示例

#### 1.清理环境：保证没有之前测试残留

rm -f /tmp/pod_ip_map.json

rm -f /var/log/my-oci-hook.log

rm -rf /sys/fs/bpf/tc/globals/*

#### 2.起三个客户端和三个服务端

kubectl apply -f iperf-client.yaml -f iperf-client-2.yaml -f iperf-client-3.yaml

kubectl apply -f iperf-server.yaml

kubectl apply -f iperf-server-2.yaml

kubectl apply -f iperf-server-3.yaml

检查每个服务端的yaml信息

    annotations:
    	enable-feature: "true"				#表示离线业务
    	enable-feature: "false"				#表示在线业务
    	bandwidth-ingress: "20mb,400mb"		#入方向带宽
    	bandwidth-egress: "20mb,200mb"		#出方向带宽
#### 3.执行打流步骤，观察每个pod的带宽限流信息

kubectl exec -it iperf-client -- iperf3 -c iperf-server -p 5201 -t 10000

kubectl exec -it iperf-client-2 -- iperf3 -c iperf-server-2 -p 5201 -t 10000

kubectl exec -it iperf-client-3 -- iperf3 -c iperf-server-3 -p 5201 -t 10000

​	查看部分map，例如

```
bpftool map dump name ips_throttle_i_ 	#观察map中的数量
```

其中iperf-server 是在线业务，iperf-server-2和iperf-server-3是离线业务，在网卡总带宽足够(5 GBytes/s以上)观察打流数据，应当是：

iperf-server ---- 在线业务，带宽无限制，带宽接近网卡上限

iperf-server-2 ---- 离线业务，带宽被限制到40mb/s

iperf-server-3 ---- 离线业务，带宽被限制到20mb/s

**停止iperf-server打流**，此时在线业务停止，离线业务带宽放宽到上限，观察到：

iperf-server-2 ---- 离线业务，带宽被限制到800mb/s

iperf-server-3 ---- 离线业务，带宽被限制到400mb/s

#### 4.依次清理pod，并观察map表中数据

kubectl delete -f iperf-server.yaml

kubectl delete -f iperf-server-2.yaml

kubectl delete -f iperf-server-3.yaml

​	查看部分map，例如

```
bpftool map dump name ips_throttle_i_ 	#观察map中的数量，此时该表应该被删除
```
